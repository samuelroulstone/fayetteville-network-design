# 🏢 Fayetteville Branch Office Network Design

> Enterprise network design for a new 50-person branch office, built as part of SNHU IT-212 (Introduction to Computer Networks).

## 📋 Overview

Designed a complete network infrastructure from the ground up for a hypothetical company opening a new branch office in Fayetteville. The project covered everything from ISP and topology selection through VLAN segmentation, remote connectivity, and quality-of-service planning for real-time applications.

## 🎯 Objective

Deliver a network design proposal that supports 50 employees across multiple departments, provides secure connectivity back to the corporate headquarters, and prioritizes traffic for video conferencing and other latency-sensitive applications.

## 🧰 Tools & Concepts

- Star network topology
- VLAN segmentation by department
- Site-to-site VPN for HQ connectivity
- Quality of Service (QoS) prioritization
- Cloud-based print management
- OSI Model reference design

## 🔍 Methodology

1. **Requirements gathering** — identified department headcounts, application needs, and growth projections.
2. **Topology selection** — chose a star topology for centralized management and easier fault isolation.
3. **ISP & connectivity planning** — evaluated ISP options for bandwidth and redundancy; designed a site-to-site VPN to securely link the branch to headquarters.
4. **VLAN design** — segmented traffic by department to reduce broadcast domains and improve security boundaries.
5. **QoS configuration** — prioritized video conferencing traffic to prevent latency/jitter issues during calls.
6. **Cloud print management** — designed a cloud-managed print solution to reduce local IT overhead.

## ✅ Key Outcomes

- Full network diagram mapping topology, VLANs, and VPN tunnel
- Departmental VLAN table with IP addressing scheme
- QoS policy recommendations for real-time traffic
- Documented rationale for ISP and topology selection tied back to business requirements
